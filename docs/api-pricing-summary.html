One-time vs. recurring — what actually repeats

Answering directly: some narration is one-time; some can never be.

Can be one-time

Narration for fixed, pre-written content

Dictation prompts, reading passages, listening clips, mock-test questions — the same script for every learner. The app already has the plumbing for this: a question_audio_url field, and an admin tool that bulk-generates audio once and stores the link.

  • If an admin ran that step for a question, every learner since has been playing the same stored file — ฿0 marginal cost per play.
  • If that step was skipped, the app quietly falls back to calling the TTS API live, every single time that question loads — full recurring cost, indefinitely.
Always recurring

Anything tied to one specific learner

Grading a response, transcribing a recording, and reading an AI-improvised follow-up question aloud all depend on content that doesn't exist until that learner acts — there's nothing to cache.

  • Grading & transcription — always this learner's own words.
  • Follow-up question narration in interactive speaking / speaking partner — the question text is generated fresh each session, so its audio can't be pre-made either.
Straight answer to "we paid once, right?": for fixed-content narration, only if it was actually pre-generated and the URL saved — that part isn't automatic today, it depends on the content-authoring step being run. For AI-improvised conversation turns, no — that audio can never be one-time by design, since the question itself is new every session.

The biggest lever isn't switching providers — it's confirming caching runs

For fixed-content flows (mock tests, mini-diagnosis, dictation, reading), narration is the largest recurring line item — roughly a third to over half of the per-attempt cost — but it's also the one that's entirely avoidable once it's generated and stored a single time. Worth confirming the admin bulk-generation step actually ran for every fixed question bank, since a skipped step means silently paying full price on every play, forever.

!

The one line item to watch regardless

ElevenLabs, the fallback voice provider, costs roughly 20–25× more per clip than Gemini's own text-to-speech — about ฿2.19 versus ฿0.10 — because it bills per character rather than per token. It only fires when Gemini TTS is unavailable or a clip exceeds the length cap, but even one cached Gemini clip is cheaper than one live ElevenLabs clip, so caching and provider choice compound.
